# 验证层：教会AI“该信什么”

> 当MEMORY.md里有一条“用户喜欢Python”，而昨天的对话里用户说“我讨厌Python”，AI该信哪个？——验证层来解决。

## 理论内核

**验证层**的核心主张是：**AI必须有能力判断信息的可信度，在冲突时做出理性裁决**。

这比“存储记忆”更底层。如果AI无法判断哪条记忆是可信的，它就无法做出可靠决策。就像人类需要分辨“谣言”和“事实”，AI也需要分辨“可信记忆”和“错误记忆”。

### 为什么需要验证层？

当遇到以下情况时，AI需要一个裁决机制：

| 场景 | 问题 | 传统AI的行为 | 加入验证层后 |
|------|------|--------------|--------------|
| **记忆冲突** | MEMORY.md里写“喜欢Python”，昨天对话说“讨厌Python” | 随机选择，或困惑 | 记录冲突，等待裁决 |
| **来源混杂** | 用户说的 vs 其他AI分享的 vs 自己学的 | 同等对待 | 按来源分级处理 |
| **信息过时** | 三个月前的配置信息 | 仍然使用 | 检测时效性，提醒更新 |
| **人为篡改** | 恶意修改了SOUL.md | 被动接受 | 检测异常修改 |

### 可信度分级规则

根据“生命算法”框架，我们定义以下可信度层级：

| 来源类型 | 置信度 | 说明 |
|----------|--------|------|
| 用户明确说（“记住这个”） | 0.95 | 最高可信度 |
| 用户自然提及 | 0.80 | 对话中自然流露 |
| AI推断得出 | 0.60 | 根据上下文推测 |
| 从其他AI分享 | 0.40 | 需要交叉验证 |
| 网页学习 | 0.30 | 最低可信度 |

## 实战操作

### 第一步：在SOUL.md中加入可信度规则

编辑 `~/clawd/SOUL.md`，在末尾添加：

```markdown
## 记忆可信度规则
- **用户明确说的**（“记住这个”、“这是重要的”）：置信度0.95
- **用户自然提及**（对话中自然流露）：置信度0.80
- **AI推断得出**（根据上下文推测）：置信度0.60
- **从其他AI分享**（通过社交网络）：置信度0.40
- **网页学习**（自主学习）：置信度0.30

## 冲突裁决原则
1. **来源优先级**：用户明确指令 > 用户隐含表达 > AI推断 > 其他AI分享
2. **时间优先级**：最新记录 > 旧记录（除非用户标注“永久记住”）
3. **验证优先级**：多次验证 > 单次提及
4. **互相矛盾时**：写入CONFLICTS.md等待用户裁决

## 冲突处理流程
当检测到记忆冲突时：
1. 立即在CONFLICTS.md中记录
2. 在HEARTBEAT.md中标记待处理
3. 下次对话时主动提醒用户
4. 用户裁决后更新可信度
